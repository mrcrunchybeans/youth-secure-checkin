## üîê Automatic Encryption Migration (NEW)

When you update to v1.0.1, database encryption migration happens **completely automatically on startup**. No manual steps needed!

**What gets encrypted:**
- Database at rest (AES-256 SQLCipher)
- Sensitive fields: names, phone numbers, addresses, notes (Fernet)

**For users:** Just add encryption keys to `.env` and restart. That's it!

```bash
# Generate encryption keys
DB_ENCRYPTION_KEY=$(openssl rand -hex 32)
FIELD_ENCRYPTION_KEY=$(python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())")

# Add to .env
echo "DB_ENCRYPTION_KEY=$DB_ENCRYPTION_KEY" >> .env
echo "FIELD_ENCRYPTION_KEY=$FIELD_ENCRYPTION_KEY" >> .env

# Restart (migration runs automatically on startup)
docker compose pull
docker compose down
docker compose up -d
```

Migration happens silently, backups are created automatically, and the app continues normally with encrypted data.

**See [DOCKER_ENCRYPTION_MIGRATION.md](https://github.com/mrcrunchybeans/youth-secure-checkin/blob/master/DOCKER_ENCRYPTION_MIGRATION.md) for full details.**

## üõ°Ô∏è Security

**Keys are unique per deployment:**
```bash
# Each instance gets its own keys
DB_ENCRYPTION_KEY=$(openssl rand -hex 32)      # 64-char hex string
FIELD_ENCRYPTION_KEY=$(python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())")  # 44-char base64
```

**Store safely:**
- ‚úÖ In `.env` (never commit to Git)
- ‚úÖ In password manager
- ‚úÖ Backup separately from database
- ‚ùå NOT in plain text in docker-compose.yml
- ‚ùå NOT hardcoded anywhere

**Backups:**
- Created automatically before encryption
- Located in `data/backups/checkin.db.backup-TIMESTAMP`
- Can be deleted after 30 days of stable operation
