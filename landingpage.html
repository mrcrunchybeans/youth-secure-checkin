A free, open-source check-in system designed for Trail Life, scouting groups, churches, schools, and youth programs. Fast family lookup by phone number, QR code checkout, and complete data privacy with self-hosting.

Built by a Trail Life troop leader, battle-tested with real families. Deploy in minutes with Docker, customize your branding, import families from CSV, and connect your calendar for automatic event population.

Key Features That Make Check-in Easy

Quick Check-in: Parents enter last 4 digits of phone, select kids, and check in – takes 10 seconds. Secure Checkout: Unique QR codes or Brother QL thermal label printing. Smart Events: Auto-import from Google Calendar/Outlook via iCal. Your Branding: Custom logo, colors, and terminology. Complete Privacy: Self-hosted, no cloud required, all data stays on your server.

Self-Hosting for Technical Users

If your ministry has tech volunteers, a home lab, or a Linux server, YouthCheckIn can be self-hosted with zero licensing costs. Complete control over your data.

Tech Stack & Requirements

Backend: Flask 3.0 (Python) | Database: SQLite (embedded, zero-config) | Frontend: Bootstrap 5.3 + vanilla JavaScript

  • Docker (recommended) - One command deployment

  • Linux - systemd service with Nginx reverse proxy

  • Windows - Run directly with Python 3.10+

    • Docker 20+ and Docker Compose
      or

Deployment Quick Start (Docker)

# Pull the latest image
$ docker pull mrcrunchybeans/youth-secure-checkin:latest

# Or try the demo mode with auto-reset
$ docker pull mrcrunchybeans/youth-secure-checkin:demo

# Run it
$ docker-compose up -d

# Access at http://localhost:5000

FAQs

Is the open-source version really free forever?

Yes. The code belongs to the community. Hosting is optional for ministries that want convenience.

Do you store any children’s data?

Not on the open-source version. For hosting customers, we store only what your ministry enters, securely.

Can we export our data?

Always, anytime.

Do we need special hardware?

To use: Any device with a web browser—phone, tablet, laptop, or desktop.

To self-host: Any Docker-capable device (Raspberry Pi 3+, NAS, VPS, home server) or Python 3.10+ on Linux/Windows.

Optional: Brother QL series thermal printer for label printing (QL-700, QL-800, QL-820NWB).

What of our organization grows?

There are no caps or limits. Growth is never penalized.
